## First of all
Recreate the UI in Figma with Claude code and Figma MCP.
Then please go over the below task list and create all the design artifacts.
Make sure the updated designs are reflected on the prototype as well.
**BIG PLUS** If you record your screen during your session.
**MILD PLUS** If you record a loom-like walkthrough where you tell us about your thinking process
and ways of solving the tasks.
### 1. Quick-create menu — the + New ▾ in the top bar
The button exists. The menu behind it doesn't. Design the dropdown so Anna can launch any of
these flows from anywhere in the product, in one click or one keystroke:
- New entry: log a transaction by hand
- New transfer: move money
- New goal: start a savings goal
- New category: add a spending category
- Connect account: link a new bank

We want to see:
- The dropdown itself: anatomy, hierarchy, states
- The first screen of each of the five flows (form, modal, side sheet... your call)
- How a confirmation/success state looks for one of the flows (your pick)
- ... and more perks you add

### 2. Search
The top bar has a search input. Design what happens when Anna uses it.
Cover at least:
- Default state
- Typing state with mixed results
- Empty state
- Keyboard interaction model
The search isn't only a finder... it's also a command palette.

### 3. Fix what's broken
The current HTML is shipped-but-not-great. As you work, find what's wrong. UX, UI, content,
consistency, accessibility, broken states, anything. And fix it in your Figma file. The dashboard
we hand to engineering should be the one you would ship.
You don't need to list every issue. In your write-up:
- Group what you found into themes
- For each theme, show one or two before/afters and explain the fix
- Flag anything you noticed but consciously chose not to fix, and why
We're looking for the eye, not the checklist.
